## Summary of Script Functions

* script.xess & script.xell: These are likely batch or configuration scripts used to automate the initialization and testing of [Intel® Xe Super Sampling (XeSS)](https://www.intel.com/content/www/us/en/developer/topic-technology/gamedev/xess.html) and Xe Low Latency (XeLL). They help developers quickly toggle quality presets or measure latency impact without manually reconfiguring the rendering engine.
* scriptz.xr: This typically refers to an OpenXR or Extended Reality (XR) script. The .xr extension is also used by developers to store [ld-scripts data](https://www.solvusoft.com/en/file-extensions/file-extension-xr/) for cross-compilation environments like Cygwin.
* script.blf: A Binary Logging File (BLF) is a specialized format for high-performance logging. In graphics development, these are used to capture:
* Frame Data: Detailed logs of render calls and GPU performance metrics.
   * Automation: Scripts for programs like [MasterWorks](https://www.youtube.com/watch?v=l-OyJPwrO7w) or automotive CAN bus analysis tools like [CANoe](https://support.vector.com/kb?id=kb_article_view&sysparm_article=KB0011536) that require small, efficient binary storage. [1]
